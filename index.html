How Does Incomplete Composite Refactoring Affect Internal Quality Attributes?


This website contains complementary data to the paper.

Abstract

Program refactoring is the process of applying code changes aimed at improving the internal code structure of a software project. Recent studies presented that developers often perform composite refactorings, which are set of two or more interrelated single refactorings. Previous studies recommended some composite refactorings to remove poor code structures, i.e, code smells, thus improving the program comprehension. Recent studies presented that composite refactorings often failed to remove the code smells. These are incomplete composite refactorings, i.e, composites that do not have one or more single refactorings recommended to fully remove a particular smell. However, those previous studies did not provide an in-depth analysis of incomplete composites. In particular, those studies do not characterize the most common incomplete composites applied to real programs, and how incomplete composite refactorings could be improving the program comprehension gradually, affecting internal quality attributes. This paper investigated the most common incomplete composites, and the effect of incomplete composite refactorings on quality attributes, regarding the program comprehension. We analyzed 353 incomplete composite refactorings in 5 software projects, two common code smells (Feature Envy and God Class), and four internal quality attributes. Our results reveal that incomplete composite refactorings with at least one Extract Method are often (71%) applied without Move Methods on smelly classes. This result suggests that the code smells are not removed and the program comprehension is not improved significantly because of the lack of Move Methods. We also have found that most incomplete composite refactorings tended to not change the internal state of the code on smelly classes. This result revealed that, despite the fact that incomplete composite refactorings did not remove the code smells, they remained the structural internal quality of the classes and the program comprehension. These results suggest that the existing refactoring tools also should evaluate if the incomplete composite refactorings are keeping the internal structure quality, while the code smells are not removed.
